# Papa Bear v1 documentation

This directory contains the durable product and architecture specifications for
Papa Bear. Historical milestone plans, release plans and superseded
implementation roadmaps have been removed. A new milestone plan will be added
separately when approved.

## Active specifications

1. `product-vision.md` — product goals and non-goals.
2. `persona-and-dialog.md` — Papa Bear role and radio behavior.
3. `world-model.md` — dynamic state, provenance, freshness and uncertainty.
4. `arma-data-contract.md` — telemetry and query contracts.
5. `privacy-and-fair-play.md` — multiplayer, privacy and authority boundaries.
6. `voice-architecture.md` — transcription, reasoning and speech architecture.

The repository-level `CHANGELOG.md` remains the historical version and change
record. It is not implementation authority for a future milestone.

Until a new milestone specification is added, changes must preserve the active
product boundaries and avoid expanding into deferred capabilities without an
explicitly approved specification.
